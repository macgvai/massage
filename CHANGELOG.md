# Changelog - Рефакторинг проекта

## ✅ Что сделано

### 1. Упрощена архитектура БД
- **Было**: Данные разбросаны по разным ключам в таблице `key_value`
- **Стало**: Вся конфигурация в одной записи с ключом `config`
- **Преимущества**: 
  - Проще работать с данными
  - Один запрос вместо множества
  - Легче делать бэкапы

### 2. Изображения теперь в БД
- **Было**: Ссылки на изображения хранились только в файловой системе
- **Стало**: Ссылки сохраняются в БД в поле `config.images`
- **Структура**:
  ```typescript
  images: {
    'about-bg': '/images/about-bg-1234567890.jpg',
    'advantages-bg': '/images/advantages-bg-1234567890.jpg',
    'diploma': '/images/diploma-1234567890.jpg',
    'master-photo': '/images/master-photo-1234567890.jpg'
  }
  ```

### 3. Упрощены API сервисы
**Файл**: `app/api/services/mainServices.ts`

- `getConfig()` - получить всю конфигурацию
- `updateConfig(config)` - обновить конфигурацию
- Убраны сложные функции мерджа и разделения данных

### 4. Рефакторинг API роутов

#### `/api/admin/config`
- Упрощен GET - просто возвращает конфигурацию
- Упрощен POST - просто сохраняет что пришло
- Убрана сложная логика мерджа

#### `/api/admin/upload`
- Добавлено сохранение ссылки на изображение в БД
- После загрузки файла автоматически обновляется `config.images`

#### `/api/admin/current-images`
- Теперь берет данные из БД, а не сканирует файловую систему
- Быстрее и надежнее

### 5. Компонент Services
**Файл**: `components/services.tsx`

- Убраны несуществующие хуки `useServices` и `useOffers`
- Данные теперь приходят через пропсы из `siteConfig`
- Исправлены все ошибки TypeScript
- Код стал проще и понятнее

### 6. Типизация
**Файл**: `types/index.ts`

- Добавлено поле `images: CurrentImages` в `SiteConfig`
- Все типы согласованы

### 7. Сиды
**Файл**: `db/seed.ts`

- Создан единый объект конфигурации с дефолтными данными
- Добавлены дефолтные пути к изображениям
- Скрипт `npm run db:seed` для быстрого заполнения БД

## 🚀 Как использовать

### Запуск проекта
```bash
npm install
npm run db:seed
npm run dev
```

### Загрузка изображений
1. Откройте админку: http://localhost:3082/admin
2. Загрузите изображение
3. Ссылка автоматически сохранится в БД
4. Изображение отобразится на сайте

### Проверка БД
```bash
sqlite3 db/db.sqlite "SELECT key FROM key_value"
# Должно вывести: config

sqlite3 db/db.sqlite "SELECT json_extract(value, '$.images') FROM key_value WHERE key='config'"
# Покажет все ссылки на изображения
```
